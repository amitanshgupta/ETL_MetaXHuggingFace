# ETL OpenEnv — Autonomous Data Cleaning Environment

A benchmark environment where an AI agent performs step-by-step data cleaning
and transformation (ETL) on messy datasets to produce clean, model-ready data.

Built for the OpenEnv Hackathon.

---

## 🧠 Core Idea

This is NOT a static pipeline.
It is a sequential decision-making environment where an agent:
- Observes a dirty dataset
- Chooses cleaning actions one step at a time
- Receives a reward signal based on data quality improvement
- Tries to match the ground truth clean dataset

## ⚙️ How It Works

### Environment Lifecycle
```python
env = ETLEnvironment(task_path="tasks/easy_missing.yaml")
obs = env.reset()           # loads dirty dataset, returns first observation

while not done:
    action = agent.act(obs) # agent picks an action
    result = env.step(action)
    obs, reward, done, info = result.observation, result.reward, result.done, result.info
```

### Action Space
The agent can apply these transformations each step:

| Action | Description |
|---|---|
| `impute_mean` | Fill nulls in a column with column mean |
| `impute_mode` | Fill nulls with most frequent value |
| `impute_constant` | Fill nulls with a fixed value |
| `drop_column` | Remove a column entirely |
| `drop_rows` | Drop rows where a column is null |
| `convert_type` | Cast a column to a target dtype |
| `normalize` | Min-max normalize a numeric column |
| `remove_duplicates` | Drop duplicate rows |
| `rename_column` | Rename a column |
| `split_column` | Split one column into multiple |
| `merge_columns` | Merge multiple columns into one |

### Observation (what the agent sees)
- Current dataset shape
- Per-column stats (dtype, missing count, unique count, sample values)
- Data quality metrics (completeness, uniqueness, validity, consistency)
- 5 sample rows
- Full action history with rewards
- Steps remaining

### Reward Signal
 - Reward = ΔQuality + Penalties

- **ΔQuality** — change in weighted quality score (completeness 35%, validity 25%, consistency 20%, uniqueness 20%)
- **Redundant action penalty** — `-0.05` if quality didn't change
- **Destructive drop penalty** — `-0.15` if >20% rows dropped
- **Invalid action penalty** — `-0.10` if action throws an error

### Tasks

| Task | Dataset | Difficulty | Corruptions |
|---|---|---|---|
| `easy_missing` | Titanic | Easy | Missing values, duplicates |
| `medium_schema` | House Prices | Medium | Type mismatches, missing, noise |
| `hard_integration` | Olist (orders + customers) | Hard | Multi-source, conflicts, duplicates, missing |

## 🚀 Setup

### 1. Clone & install
```bash
git clone <repo-url>
cd etl-openenv
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
pip install -r requirements.txt
```

### 2. Download raw datasets
Place these files in `data/raw/`:

| File | Source |
|---|---|
| `titanic.csv` | [Kaggle — Titanic](https://www.kaggle.com/datasets/yasserh/titanic-dataset) |
| `house_prices_train.csv` | [Kaggle — House Prices](https://www.kaggle.com/datasets/lespin/house-prices-dataset) (use `train.csv`) |
| `olist_orders_dataset.csv` | [Kaggle — Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) |
| `olist_customers_dataset.csv` | same Olist download |
| `olist_order_items_dataset.csv` | same Olist download |

### 3. Generate datasets
```bash
python -m data.generators.dataset_loader   # creates data/clean/
python -m data.generators.corruption       # creates data/dirty/
```

Expected output:
    Titanic clean: (891, 12)
    House prices clean: (1460, 13)
    Olist clean: (113425, 9)
    Titanic dirty: (936, 12)
    House prices dirty: (1460, 13)
    Olist dirty: (122499, 9)

## 🧩 Key Design Decisions

- **No in-place mutation** — every action copies the DataFrame before modifying, safe for reward delta computation
- **Deterministic corruption** — `seed=42` everywhere, results are always reproducible
- **Dense rewards** — agent always gets a signal, no sparse reward problem
- **Typed everything** — Pydantic models for all inputs/outputs, no raw dicts crossing module boundaries
- **YAML-driven tasks** — adding a new task = new YAML file, zero code changes
